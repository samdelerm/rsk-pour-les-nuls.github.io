Python : Objets & Classes

La bibliothèque RSK est entièrement construite autour des objets. Ce chapitre t'explique ce que ça veut dire et comment t'en servir.

1. Qu'est-ce qu'un objet ?

En Python, un objet est une variable spéciale qui regroupe à la fois des données (qu'on appelle attributs) et des actions (qu'on appelle méthodes).

Imagine un robot physique : il a une position, une orientation, une couleur (→ attributs), et il peut se déplacer, tirer, klaxonner (→ méthodes). C'est exactement ainsi que fonctionne l'objet client.green1 dans RSK.

Attributs (les données)
client.green1.position
client.green1.orientation
client.green1.color
client.ball

On y accède avec un point, sans parenthèses.

Méthodes (les actions)
client.green1.goto(...)
client.green1.kick()
client.green1.control(...)
client.green1.leds(...)

On les appelle avec un point + parenthèses.

La règle simple : objet.attribut (sans parenthèses) = lire une donnée. objet.methode() (avec parenthèses) = faire une action.

2. Définir une classe

Une classe est le plan de fabrication d'un objet. On la définit avec le mot-clé class, et on crée des instances avec la classe comme si c'était une fonction.

ma_premiere_classe.py
class Robot:
    """Représente un robot de foot simplifié."""

    def __init__(self, couleur, numero):
        """
        __init__ est le constructeur : il est appelé quand on crée un Robot.
        self représente l'objet lui-même.
        """
        self.couleur  = couleur    # attribut
        self.numero   = numero     # attribut
        self.position = (0.0, 0.0) # attribut avec valeur par défaut

    def se_presenter(self):
        """Méthode : affiche les infos du robot."""
        print(f"Je suis le robot {self.couleur} n°{self.numero}")
        print(f"  Position : {self.position}")


# ── Créer des instances ──────────────────────────────────
r1 = Robot("vert", 1)
r2 = Robot("vert", 2)
bleu = Robot("bleu", 1)

# ── Utiliser les attributs ────────────────────────────────
print(r1.couleur)       # vert
print(r2.numero)        # 2

# ── Modifier un attribut ──────────────────────────────────
r1.position = (0.5, 0.2)
print(r1.position)      # (0.5, 0.2)

# ── Appeler une méthode ───────────────────────────────────
r1.se_presenter()
# → Je suis le robot vert n°1
# →   Position : (0.5, 0.2)

3. Les attributs

Les attributs sont les données stockées dans l'objet. On les crée dans __init__ avec self.nom = valeur et on y accède depuis l'extérieur avec objet.nom.

attributs_rsk.py
import rsk

with rsk.Client(host='localhost', key='') as client:
    # ── Attributs du robot ─────────────────────────────────
    pos   = client.green1.position      # tuple (x, y) ou None
    angle = client.green1.orientation   # float en radians ou None
    pose  = client.green1.pose          # tuple (x, y, angle) ou None
    age   = client.green1.age()         # secondes depuis la dernière mise à jour

    # ── Attribut de la balle ──────────────────────────────
    balle = client.ball     # tuple (x, y) ou None (si pas visible)

    # ── Utiliser les attributs ────────────────────────────
    if pos is not None:
        x, y = pos              # décomposer le tuple
        print(f"Robot vert 1 : x={x:.2f}, y={y:.2f}")

    if balle is not None:
        bx = balle[0]           # accès par index
        by = balle[1]
        print(f"Balle : x={bx:.2f}, y={by:.2f}")

Différence entre attribut et méthode — piège classique !

attribut_vs_methode.py
import rsk

with rsk.Client(host='localhost', key='') as client:
    # ✅ CORRECT : position est un ATTRIBUT → pas de ()
    pos = client.green1.position
    print(pos)           # (0.46, 0.0)

    # ✅ CORRECT : goto() est une MÉTHODE → avec ()
    client.green1.goto((0.5, 0.0, 0))

    # ❌ ERREUR : position n'est pas une méthode
    # pos = client.green1.position()   # TypeError !

    # ❌ ERREUR : goto n'est pas un attribut
    # pos = client.green1.goto         # ne fait rien, objet non appelé

4. Les méthodes

Une méthode est une fonction définie dans une classe. Elle reçoit toujours self comme premier paramètre (représentation de l'objet lui-même).

methodes.py
import math

class Robot:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def distance_a(self, autre_x, autre_y):
        """Calcule la distance vers un point."""
        return math.hypot(autre_x - self.x, autre_y - self.y)

    def angle_vers(self, cible_x, cible_y):
        """Calcule l'angle (en rad) à regarder pour viser un point."""
        return math.atan2(cible_y - self.y, cible_x - self.x)

    def est_pres_de(self, cible_x, cible_y, seuil=0.1):
        """Retourne True si le robot est à moins de seuil mètres."""
        return self.distance_a(cible_x, cible_y) < seuil


# Test
r = Robot(0.5, 0.0)
print(r.distance_a(0.0, 0.0))       # 0.5
print(r.angle_vers(-0.9, 0.0))      # ≈ π (regarder à gauche)
print(r.est_pres_de(0.55, 0.0))     # True (distance ≈ 0.05)

Héritage — comment RSK l'utilise

L'héritage permet de créer une nouvelle classe basée sur une existante. Dans RSK, ClientRobot hérite de ClientTracked :

heritage.py
# ── Classe parente ────────────────────────────────────────
class Vehicule:
    def __init__(self, vitesse_max):
        self.vitesse_max = vitesse_max
        self.vitesse = 0

    def accelerer(self, v):
        self.vitesse = min(v, self.vitesse_max)

    def infos(self):
        print(f"Vitesse : {self.vitesse} / {self.vitesse_max}")


# ── Classe fille (hérite de Vehicule) ─────────────────────
class RobotFoot(Vehicule):
    def __init__(self, vitesse_max, couleur):
        super().__init__(vitesse_max)   # appelle le __init__ parent
        self.couleur = couleur
        self.buts = 0

    def marquer(self):
        self.buts += 1
        print(f"But ! ({self.buts} au total)")

    def infos(self):                    # redéfinit la méthode parente
        super().infos()                 # appelle quand même la version parente
        print(f"Couleur : {self.couleur}, Buts : {self.buts}")


# ── Utilisation ───────────────────────────────────────────
r = RobotFoot(1.5, "vert")
r.accelerer(1.0)    # méthode héritée de Vehicule
r.marquer()         # méthode propre à RobotFoot
r.infos()

5. Le mot-clé with

with est un gestionnaire de contexte. Il s'assure qu'une ressource est correctement ouverte et automatiquement fermée à la fin du bloc, même en cas d'erreur.

Dans RSK, with rsk.Client(...) as client: ouvre la connexion au serveur et la ferme proprement quand le programme se termine.

with_explique.py
# ── Exemple avec un fichier ───────────────────────────────
with open("data.txt", "w") as fichier:
    fichier.write("Position : 0.5, 0.2\n")
# Ici, le fichier est AUTOMATIQUEMENT fermé
# même si une exception survient à l'intérieur du bloc.

# ── Exemple avec RSK ──────────────────────────────────────
import rsk

with rsk.Client(host='localhost', key='') as client:
    # 'client' est disponible ici
    while True:
        if client.ball is not None:
            bx, by = client.ball
            client.green1.goto((bx + 0.05, by, 3.14), wait=False)
# Quand le programme se termine (Ctrl+C ou fin du bloc),
# la connexion RSK est correctement fermée.

# ── Sans with (déconseillé) ───────────────────────────────
client = rsk.Client(host='localhost', key='')
client.__enter__()    # ouverture manuelle (pas propre !)
# ... code ...
client.__exit__()     # fermeture manuelle → risque d'oubli

Tout programme RSK commence par with rsk.Client(...) as client:. Le reste du code (la boucle while True et les fonctions) est indenté à l'intérieur de ce bloc.

6. Exceptions : try / except

Une exception est une erreur qui survient pendant l'exécution. Sans la gérer, elle fait planter ton programme. Avec try/except, tu peux attraper l'erreur et continuer à tourner.

try_except.py
import rsk
import math
from math import radians

def Attaquant(xball, yball, xg1, yg1):
    try:
        # ⚠️ Division possible par zéro si xball == xg1
        angle = math.atan((yball - yg1) / (xball - xg1))
        client.green1.goto((xball + 0.05, yball, radians(180) + angle), wait=False)
        client.green1.kick()

    except ZeroDivisionError:
        # Si xball == xg1, atan(dy/0) lève une erreur
        print("Division par zéro — robot sur la même ligne que la balle")
        client.green1.goto((xball + 0.05, yball, radians(180)), wait=False)

    except rsk.client.ClientError as e:
        # Erreur de communication avec le serveur RSK
        print(f"Erreur RSK : {e}")

    except Exception as e:
        # Attraper n'importe quelle autre erreur
        print(f"Erreur inattendue : {e}")


with rsk.Client(host='localhost', key='') as client:
    while True:
        try:
            xball = client.ball[0]
            yball = client.ball[1]
            xg1   = client.green1.position[0]
            yg1   = client.green1.position[1]
            Attaquant(xball, yball, xg1, yg1)
        except:
            pass    # Ignorer toutes les erreurs (ball=None, etc.)

Le bloc except: pass (attraper toutes les erreurs en silence) est utilisé dans les scripts RSK débutants pour éviter les plantages sur None. C'est pratique mais cache les vrais bugs — préfère vérifier explicitement avec if client.ball is not None:.

7. La classe Client de RSK en détail

Maintenant que tu connais les objets, voici comment RSK les utilise concrètement. rsk.Client est une classe dont l'instance s'appelle client. Elle contient des objets ClientRobot pour chaque robot.

client_rsk_detail.py
import rsk

with rsk.Client(host='localhost', key='') as client:
    # client est une instance de la classe rsk.Client
    # Elle CONTIENT d'autres objets :

    # ── Les robots (instances de ClientRobot) ─────────────
    r_v1 = client.green1   # Robot vert n°1
    r_v2 = client.green2   # Robot vert n°2
    r_b1 = client.blue1    # Robot bleu n°1
    r_b2 = client.blue2    # Robot bleu n°2

    # ── La balle (tuple ou None) ─────────────────────────
    balle = client.ball    # (x, y) ou None

    # ── Attributs d'un robot ─────────────────────────────
    print(r_v1.position)      # (x, y) ou None
    print(r_v1.orientation)   # angle en radians ou None
    print(r_v1.pose)          # (x, y, angle) ou None
    print(r_v1.color)         # "green"
    print(r_v1.number)        # 1

    # ── Méthodes d'un robot ──────────────────────────────
    r_v1.goto((0.3, 0.0, 3.14))      # aller en (0.3, 0, π)
    r_v1.kick(1.0)                    # tirer à pleine puissance
    r_v1.control(0.3, 0.0, 0.0)      # avancer à 0.3 m/s
    r_v1.leds(0, 255, 0)             # LEDs vertes
    r_v1.beep(440, 200)              # La (440 Hz) pendant 200 ms

Schéma de la hiérarchie

rsk.Client (la connexion au serveur)
 ├── client.ball → tuple (x, y) ou None
 ├── client.green1 → objet ClientRobot
 │   ├── .position → (x, y)
 │   ├── .orientation → angle en radians
 │   ├── .goto(target, wait) → bool
 │   └── .kick(power)
 ├── client.green2 → même structure
 ├── client.blue1 → même structure
 └── client.blue2 → même structure
