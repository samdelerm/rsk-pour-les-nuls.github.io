📖 Chapitre 5

Tactiques : Défenseur & Attaquant

Analyse complète ligne par ligne des deux programmes d'équipe. Comprends chaque choix de code et comment les adapter.

1. Vue d'ensemble — deux robots, deux rôles

L'équipe verte comprend deux robots. Pour maximiser les chances de victoire, chacun a un rôle précis :

🛡️ Green 2 — Défenseur
  • Se place devant son propre but
  • Suit la balle verticalement (axe Y)
  • Tire pour dégager si la balle pénètre dans la zone dangereuse
  • Reste dans les limites Y [−0.30, +0.30] devant le but
⚔️ Green 1 — Attaquant
  • Se positionne juste derrière la balle
  • Oriente son tir vers le but adverse avec atan
  • Tire dès qu'il est en position
  • Ajuste sa trajectoire si la balle est devant lui

La stratégie dépend de quel côté le vert défend. "vert vers−" = vert défend à droite (x≈+0.92), attaque vers x négatif. "vert vers+" = vert défend à gauche (x≈−0.92), attaque vers x positif.

2. Le Défenseur — logique détaillée

Voici la logique complète du défenseur, avec l'explication de chaque décision :

defenseur_commente.py (version vers−, but vert à droite)
def Defenseur(xball, yball):
    """
    Green2 défend le but droit (x ≈ +0.92).
    Règle : se placer à x=0.80 en face de la balle,
    dans les limites Y du but [−0.30, +0.30].
    """
    if client.ball is not None:

        # ── Cas 1 : la balle approche (est à gauche du défenseur) ──
        # Et elle est dans la zone Y du but
        if xball < client.green2.position[0] and yball < 0.30 and yball > -0.30:
            # Se mettre à x=0.80 (devant le but) au niveau de la balle
            # radians(180) → le robot regarde VERS LA GAUCHE (vers la balle, vers l'adversaire)
            client.green2.goto((0.80, yball, radians(180)), wait=False)

        # ── Cas 2 : la balle est dans la zone de but (urgence !) ──
        if yball > -0.30 and yball < 0.30 and xball > 0.60 and xball < 0.92:
            # Se placer juste derrière la balle et TIRER pour dégager
            client.green2.goto((xball + 0.07, yball, radians(180)), wait=False)
            client.green2.kick()
            client.green2.kick()    # double kick pour plus de puissance

        else:
            # ── Cas 3 : balle hors de la zone Y ──
            # Aller au coin le plus proche pour rester couvert
            if yball >= 0.30:
                # Balle en haut → se caler en haut du but
                client.green2.goto((0.80, 0.30, radians(180)))
            else:
                # Balle en bas → se caler en bas du but
                if yball <= -0.30:
                    client.green2.goto((0.80, -0.30, radians(180)))

Décortiquons chaque condition

L.9–11
Cas 1 — la balle approche dans la zone centrale. xball < position[0] : la balle est plus à gauche que le défenseur (elle se dirige vers son but). On suit la balle sur l'axe Y avec wait=False pour rester réactif.
L.14–17
Cas 2 — URGENCE : la balle est dans la zone de but. xball > 0.60 et < 0.92 : la balle est entre 60 cm et 92 cm sur X. C'est la zone de but. On place le robot juste derrière la balle (xball + 0.07) et on tire deux fois pour dégager.
L.20–25
Cas 3 — la balle est hors de la zone Y du but. Si la balle est très haute (y ≥ 0.30) ou très basse (y ≤ −0.30), le défenseur va se caler aux extrémités du but : (0.80, ±0.30). Il reste ainsi toujours dans sa zone de protection.

3. L'Attaquant — logique détaillée

attaquant_commente.py (version vers−)
import math

def Attaquant(xball, yball, xg1, yg1, atb1):
    """
    Green1 attaque vers le but bleu (x ≈ −0.92).
    atb1 = math.atan((yball - yg1) / (xball - xg1))
         → angle entre le robot et la balle
    """
    try:
        if client.ball is not None:

            # ── La balle est devant le robot (à gauche) ──
            if xball < xg1:
                # Se placer légèrement derrière la balle sur X
                # en ajoutant l'angle d'approche (atb1) pour orienter vers le but
                if client.green1.goto(
                    (xball + 0.05, yball, radians(180) + atb1),
                    wait=False
                ):
                    # True = arrivé → TIRER !
                    client.green1.kick(1)
                    client.green1.kick()

            else:
                # ── La balle est DERRIÈRE le robot ──
                # Approche légèrement décalée en Y (−0.15) pour se repositionner
                # sans percuter la balle dans la mauvaise direction
                if client.green1.goto(
                    (xball + 0.05, yball - 0.15, radians(180) + atb1),
                    wait=False
                ):
                    client.green1.kick()
                    client.green1.kick(1)

    except rsk.client.ClientError as e:
        print(e)
atb1
L'angle de tir. math.atan((yball - yg1) / (xball - xg1)) calcule l'angle entre le robot et la balle. Ajouté à radians(180), ça oriente le robot pour qu'il regarde vers la balle puis vers le but adverse.
L.13–19
Balle devant → approche directe. Si xball < xg1 (la balle est à gauche du robot), le robot fonce dessus avec un offset de 5 cm. Quand goto() retourne True, il tire.
L.23–29
Balle derrière → contournement. Si la balle est derrière le robot (xball > xg1), approcher frontalement la percuterait dans la mauvaise direction. On se décale de 15 cm sur Y pour d'abord se repositionner correctement.

4. Code complet — vert attaque vers− (but vert à droite)

C'est le fichier green vers-.py du dossier "vert basique", annoté et nettoyé pour une meilleure lisibilité.

green_vers_moins.py — version complète annotée
from math import radians
import math
import rsk

# ╔══════════════════════════════════════════════════════════════╗
# ║  VERT VERS −                                                  ║
# ║  Le but VERT est à droite (+X ≈ +0.92)                      ║
# ║  L'équipe VERTE attaque vers la gauche (−X)                  ║
# ║  Le but BLEU est à gauche (−X ≈ −0.92)                      ║
# ╚══════════════════════════════════════════════════════════════╝


def Defenseur(xball, yball):
    """Green2 défend le but droit (x ≈ 0.92)."""
    if client.ball is not None:

        # Balle qui approche dans la zone centrale → suivre sur Y
        if xball < client.green2.position[0] and yball < 0.30 and yball > -0.30:
            client.green2.goto((0.80, yball, radians(180)), wait=False)

        # Balle dans la zone de but → dégager en tirant
        if yball > -0.30 and yball < 0.30 and xball > 0.60 and xball < 0.92:
            client.green2.goto((xball + 0.07, yball, radians(180)), wait=False)
            client.green2.kick()
            client.green2.kick()

        else:
            # Balle hors zone → se caler aux extrémités du but
            if yball >= 0.30:
                client.green2.goto((0.80, 0.30, radians(180)))
            else:
                if yball <= -0.30:
                    client.green2.goto((0.80, -0.30, radians(180)))


def Attaquant(xball, yball, xg1, yg1, atb1):
    """Green1 attaque vers le but gauche (x ≈ −0.92)."""
    try:
        if client.ball is not None:
            if xball < xg1:
                # Balle devant → approche directe et tir
                if client.green1.goto((xball + 0.05, yball, radians(180) + atb1),
                                      wait=False):
                    client.green1.kick(1)
                    client.green1.kick()
            else:
                # Balle derrière → contournement par le bas
                if client.green1.goto((xball + 0.05, yball - 0.15, radians(180) + atb1),
                                      wait=False):
                    client.green1.kick()
                    client.green1.kick(1)

    except rsk.client.ClientError as e:
        print(e)


# ── CONNEXION ET BOUCLE PRINCIPALE ────────────────────────────────
with rsk.Client(host='10.2.162.102', key='') as client:
    while True:
        try:
            # Récupère les données en temps réel
            xball = client.ball[0]
            yball = client.ball[1]
            xg1   = client.green1.position[0]
            yg1   = client.green1.position[1]

            # Calcul de l'angle du robot vers la balle
            # ⚠️ Division possible par zéro si xball == xg1 !
            atb1 = math.atan((yball - yg1) / (xball - xg1))

            # Exécution des deux stratégies
            Defenseur(xball, yball)
            Attaquant(xball, yball, xg1, yg1, atb1)

        except:
            pass    # Ignore les erreurs transitoires (ball=None, etc.)

5. Code complet — vert attaque vers+ (but vert à gauche)

C'est le fichier green vers+.py. La logique est symétrique : vert défend à gauche, attaque vers la droite.

green_vers_plus.py — version complète annotée
from math import radians
import math
import rsk

# ╔══════════════════════════════════════════════════════════════╗
# ║  VERT VERS +                                                  ║
# ║  Le but VERT est à gauche (−X ≈ −0.92)                      ║
# ║  L'équipe VERTE attaque vers la droite (+X)                  ║
# ║  Le but BLEU est à droite (+X ≈ +0.92)                      ║
# ╚══════════════════════════════════════════════════════════════╝


def Defenseur(xball, yball):
    """Green2 défend le but gauche (x ≈ −0.92)."""
    if client.ball is not None:

        # Balle qui approche dans la zone centrale → suivre sur Y
        # (balle est à DROITE du défenseur maintenant → xball > position[0])
        if xball > client.green2.position[0] and yball < 0.30 and yball > -0.30:
            # radians(0) → le robot regarde vers la DROITE
            client.green2.goto((-0.80, yball, radians(0)), wait=False)

        # Balle dans la zone de but gauche → dégager
        if yball >= -0.30 and yball <= 0.30 and xball <= -0.50 and xball >= -0.94:
            client.green2.goto((xball - 0.10, yball, radians(0)), wait=False)
            client.green2.kick(1)
            client.green2.kick()

        else:
            # Balle hors zone → se caler aux extrémités du but
            if yball > 0.30:
                client.green2.goto((-0.80, 0.30, radians(0)))
            else:
                if yball < -0.30:
                    client.green2.goto((-0.80, -0.30, radians(0)))


def Attaquant(xball, yball, xg1, atb1):
    """Green1 attaque vers le but droit (x ≈ +0.92)."""
    try:
        if client.ball is not None:
            if xball > xg1:
                # Balle devant (à droite) → approche directe et tir
                if client.green1.goto((xball - 0.08, yball, atb1), wait=False):
                    client.green1.kick()
                    client.green1.kick()
            else:
                # Balle derrière → contournement
                if client.green1.goto((xball - 0.08, yball - 0.15, atb1), wait=False):
                    client.green1.kick()
                    client.green1.kick()

    except rsk.client.ClientError as e:
        print(e)


# ── CONNEXION ET BOUCLE PRINCIPALE ────────────────────────────────
with rsk.Client(host='10.2.162.102', key='') as client:
    while True:
        try:
            xball = client.ball[0]
            yball = client.ball[1]
            xg1   = client.green1.position[0]
            yg1   = client.green1.position[1]

            atb1 = math.atan((yball - yg1) / (xball - xg1))

            Defenseur(xball, yball)
            Attaquant(xball, yball, xg1, atb1)

6. Conseils & pièges courants

⚠️ Piège 1 — Division par zéro dans atan

math.atan((yball - yg1) / (xball - xg1)) plante si xball == xg1. Solution : utiliser math.atan2(yball - yg1, xball - xg1) qui ne divise jamais.

correction_atan.py
import math

# ❌ Peut planter si xball == xg1
atb1 = math.atan((yball - yg1) / (xball - xg1))

# ✅ atan2 : jamais de division par zéro
atb1 = math.atan2(yball - yg1, xball - xg1)

⚠️ Piège 2 — Accéder à ball[0] quand ball est None

client.ball[0] plante si client.ball est None (balle non visible). C'est pour ça que les codes d'équipe ont un try/except ou une vérification if client.ball is not None.

gestion_none.py
# ❌ ERREUR si ball est None
xball = client.ball[0]   # TypeError: 'NoneType' object is not subscriptable

# ✅ Solution 1 : vérification explicite (recommandée)
if client.ball is not None:
    xball = client.ball[0]
    yball = client.ball[1]
    # ... utiliser xball et yball ...

# ✅ Solution 2 : try/except (tolérant mais cache les bugs)
try:
    xball = client.ball[0]
    yball = client.ball[1]
except:
    pass

# ✅ Solution 3 : décompression avec vérification
balle = client.ball
if balle:
    xball, yball = balle

⚠️ Piège 3 — wait=True dans une boucle réactive

Si tu oublies wait=False, ton programme attend que le robot arrive à destination avant de continuer — pendant ce temps, la balle peut avoir bougé et le défenseur ne répond plus !

💡 Amélioration — Version robuste et propre

version_amelioree.py
"""
Version améliorée du programme vert vers−
Corrections :
  - atan2 à la place de atan (évite division par zéro)
  - Vérification is not None avant d'utiliser ball
  - try/except plus ciblé
"""
from math import radians
import math
import rsk


def Defenseur(client, xball, yball):
    """Green2 défend le but droit (x ≈ +0.92)."""
    pos2 = client.green2.position
    if pos2 is None:
        return  # Robot non visible, on ne fait rien

    if abs(yball) <= 0.30:
        # Zone centrale : suivre la balle sur Y
        if xball < pos2[0]:
            client.green2.goto((0.80, yball, radians(180)), wait=False)

        # Zone de but : dégager !
        if 0.60 < xball < 0.92:
            client.green2.goto((xball + 0.07, yball, radians(180)), wait=False)
            client.green2.kick()
            client.green2.kick()
    else:
        # Hors zone : se caler au coin le plus proche
        coin_y = 0.30 if yball > 0 else -0.30
        client.green2.goto((0.80, coin_y, radians(180)), wait=False)


def Attaquant(client, xball, yball, xg1, yg1):
    """Green1 attaque vers le but bleu (x ≈ −0.92)."""
    try:
        # ✅ atan2 : pas de division par zéro
        atb1 = math.atan2(yball - yg1, xball - xg1)

        if xball < xg1:
            if client.green1.goto((xball + 0.05, yball, radians(180) + atb1),
                                   wait=False):
                client.green1.kick(1)
                client.green1.kick()
        else:
            if client.green1.goto((xball + 0.05, yball - 0.15, radians(180) + atb1),
                                   wait=False):
                client.green1.kick()
                client.green1.kick(1)

    except rsk.client.ClientError as e:
        print(f"Erreur RSK : {e}")


# ── BOUCLE PRINCIPALE ─────────────────────────────────────────────
with rsk.Client(host='10.2.162.102', key='') as client:
    while True:
        # Vérification propre avant tout
        if client.ball is None:
            continue
        if client.green1.position is None or client.green2.position is None:
            continue

        xball = client.ball[0]
        yball = client.ball[1]
        xg1   = client.green1.position[0]
        yg1   = client.green1.position[1]

        Defenseur(client, xball, yball)
        Attaquant(client, xball, yball, xg1, yg1)

Idées d'améliorations supplémentaires :
— Passe client en paramètre des fonctions (comme dans la version améliorée) pour éviter les variables globales.
— Utilise math.atan2 au lieu de math.atan.
— Ajoute des LEDs de signalisation (leds(255,0,0) en danger, leds(0,255,0) en attaque).
— Implémente un historique de position pour anticiper où ira la balle.
