RSK : Contrôler les robots

Connexion, déplacement, tir, vitesse, LEDs — toutes les commandes disponibles dans la bibliothèque RSK expliquées avec des exemples concrets.

1. Se connecter au serveur

Tout programme RSK commence par with rsk.Client(...) as client:. Les paramètres dépendent de si tu utilises le simulateur ou de vrais robots.

rsk.Client(host, key='')
Crée une connexion au serveur RSK (simulateur ou physique).
hoststr'localhost' pour le simulateur local, ou l'IP de l'arbitre (ex : '10.2.162.102')
keystrClé d'authentification (souvent vide '' en cours)
connexion.py
import rsk

# ── Simulateur (sur ton ordi) ──────────────────────────────
# Lance d'abord : python -m rsk.simulator
with rsk.Client(host='localhost', key='') as client:
    print("Connecté au simulateur !")
    # ... ton code ici ...

# ── Robot réel (IP de l'arbitre WiFi) ─────────────────────
with rsk.Client(host='10.2.162.102', key='') as client:
    print("Connecté aux robots réels !")
    # ... ton code ici ...

# Après le bloc "with", la connexion est automatiquement fermée.
# Le programme principal s'arrête proprement.

Pour travailler chez toi sans robots : lance le simulateur RSK avec python -m rsk.simulator dans un terminal, puis exécute ton script dans un autre terminal. Tu verras les robots se déplacer en temps réel dans la fenêtre du simulateur.

2. Connaître la position d'un robot

Chaque robot expose sa position via des attributs (rappel : sans parenthèses !). Ces valeurs sont mises à jour en permanence par le système de vision.

robot.position
Retourne un tuple (x, y) en mètres, ou None si le robot n'est pas visible.
robot.orientation
Retourne l'angle en radians, ou None si non visible.
robot.pose
Retourne le tuple complet (x, y, angle), ou None.
robot.age()
Secondes depuis la dernière détection. Valeur élevée = robot peut-être caché.
position_robot.py
import rsk
import math

with rsk.Client(host='localhost', key='') as client:
    while True:
        # ── Position (x, y) ───────────────────────────────
        pos = client.green1.position
        if pos is not None:
            x, y = pos
            print(f"Vert 1 → x={x:.3f} m, y={y:.3f} m")

        # ── Orientation (angle) ───────────────────────────
        angle = client.green1.orientation
        if angle is not None:
            deg = math.degrees(angle)
            print(f"Vert 1 → angle={angle:.3f} rad ({deg:.1f}°)")

        # ── Pose complète (x, y, angle) ───────────────────
        pose = client.green1.pose
        if pose is not None:
            x, y, theta = pose

        # ── X et Y séparément ─────────────────────────────
        # Dans les codes d'équipe, on accède souvent directement :
        xg1 = client.green1.position[0]   # x du robot vert 1
        yg1 = client.green1.position[1]   # y du robot vert 1
        xg2 = client.green2.position[0]   # x du robot vert 2

3. goto() — Se déplacer vers un point

C'est la commande la plus utilisée dans RSK. Tu donnes une destination (x, y, angle) et le robot s'y rend en évitant (parfois) les obstacles.

robot.goto(target, wait=True, avoid_obstacles=False)
Envoie le robot vers la position cible. Retourne True quand le robot est arrivé.
targettupleTuple (x, y, angle) ou fonction retournant ce tuple
waitboolTrue = bloque jusqu'à l'arrivée. False = retourne immédiatement (mode réactif)
avoid_obstaclesboolTrue = essaie d'éviter les autres robots
goto.py
import rsk
from math import radians

with rsk.Client(host='localhost', key='') as client:

    # ── wait=True : bloque jusqu'à l'arrivée ─────────────
    # ⚠️ La boucle while s'arrête ici jusqu'à ce que le robot arrive !
    client.green1.goto((0.3, 0.0, radians(180)))
    print("Arrivé !")

    # ── wait=False : retourne immédiatement ───────────────
    # ✅ Recommandé dans une boucle while True pour rester réactif
    while True:
        if client.ball is not None:
            bx, by = client.ball

            # Se placer derrière la balle (x + offset) pour tirer
            arrive = client.green1.goto(
                (bx + 0.08, by, radians(180)),
                wait=False
            )

            # arrive == True quand le robot est à destination
            if arrive:
                client.green1.kick()

Résumé : Dans une boucle while True, utilise toujours wait=False. Utilise wait=True seulement pour une séquence linéaire (robot qui exécute des étapes l'une après l'autre).

Suivre une cible dynamique

Si la cible change en permanence (comme la balle), passe une fonction lambda comme target :

goto_dynamique.py
import rsk
from math import radians

with rsk.Client(host='localhost', key='') as client:
    while True:
        if client.ball is not None:
            # Lambda recalculée à chaque appel → cible toujours à jour
            client.green1.goto(
                lambda: (client.ball[0] + 0.08, client.ball[1], radians(180)),
                wait=False
            )

4. kick() — Tirer dans la balle

robot.kick(power=1.0)
Active le mécanisme de tir du robot. Le tir s'effectue vers l'avant du robot.
powerfloatPuissance du tir, de 0 (faible) à 1 (maximum). Par défaut : 1.
kick.py
import rsk
from math import radians

with rsk.Client(host='localhost', key='') as client:
    while True:
        if client.ball is not None:
            bx, by = client.ball
            xg2 = client.green2.position[0]

            # ── Se placer derrière la balle PUIS tirer ────
            if client.green1.goto((bx + 0.05, by, radians(180)), wait=False):
                # True = robot arrivé → TIRER !
                client.green1.kick(1)    # tir à pleine puissance

            # ── Tir dégagement (défenseur) ────────────────
            # Balle dans la zone de but → dégager immédiatement
            if bx > 0.60 and abs(by) < 0.30:
                client.green2.goto((bx + 0.07, by, radians(180)), wait=False)
                client.green2.kick()     # kick() sans argument = power=1
                client.green2.kick()     # double kick pour plus de force

            # ── Tir précis (attaquant, puissance partielle) ─
            client.green1.kick(0.5)   # tir à mi-puissance pour la précision

Le robot doit être orienté dans la bonne direction pour que le tir soit efficace. goto((bx + offset, by, angle)) place le robot derrière la balle, faisant face à la direction de tir.

Pourquoi double kick ?

double_kick.py
# Dans les codes d'équipe, on voit souvent :
client.green1.kick(1)
client.green1.kick()

# Pourquoi deux fois ?
# → La première commande "prépare" le système de tir.
# → La deuxième déclenche le tir.
# → Sur certains robots ou selon le timing, doubler
#   la commande améliore la fiabilité du tir.
# → C'est une bonne pratique quand le robot est en position.

5. control() — Contrôle en vitesse

Contrairement à goto() qui gère automatiquement le déplacement, control() te laisse contrôler les vitesses directement. C'est le contrôle bas-niveau.

robot.control(dx, dy, dturn)
Contrôle les vitesses du robot dans son repère propre (pas le repère du terrain).
dxfloatVitesse vers l'avant du robot [m/s]. Positif = avance, négatif = recule.
dyfloatVitesse latérale [m/s]. Positif = va à gauche du robot.
dturnfloatVitesse de rotation [rad/s]. Positif = tourne à gauche.
control.py
import rsk

with rsk.Client(host='localhost', key='') as client:

    # ── Avancer tout droit à 0.3 m/s ─────────────────────
    client.green1.control(0.3, 0.0, 0.0)

    # ── Reculer ───────────────────────────────────────────
    client.green1.control(-0.2, 0.0, 0.0)

    # ── Se déplacer latéralement ──────────────────────────
    client.green1.control(0.0, 0.2, 0.0)

    # ── Tourner sur place (sens anti-horaire) ─────────────
    client.green1.control(0.0, 0.0, 1.0)

    # ── S'arrêter ─────────────────────────────────────────
    client.green1.control(0.0, 0.0, 0.0)

    # ⚠️ ATTENTION : control() est dans le repère du ROBOT,
    # pas dans le repère du terrain !
    # Si le robot regarde vers la gauche, dx=0.3 le fait aller VERS LA GAUCHE,
    # pas vers la droite !

control() est utile pour des comportements fins (glissement latéral, freinage) mais goto() est généralement plus simple et plus facile à utiliser pour les déplacements vers un point.

6. leds() & beep() — Effets et son

robot.leds(r, g, b)
Change la couleur des LEDs du robot.
rintCanal rouge, de 0 à 255
gintCanal vert, de 0 à 255
bintCanal bleu, de 0 à 255
robot.beep(frequency, duration)
Fait émettre un son au robot.
frequencyintFréquence en Hz (ex : 440 = La, 523 = Do, 880 = La aigu)
durationintDurée en millisecondes (ex : 200 = court, 1000 = 1 seconde)
leds_beep.py
import rsk
import time

with rsk.Client(host='localhost', key='') as client:

    # ── LEDs de différentes couleurs ──────────────────────
    client.green1.leds(255, 0,   0)    # rouge
    client.green1.leds(0,   255, 0)    # vert
    client.green1.leds(0,   0,   255)  # bleu
    client.green1.leds(255, 127, 0)    # orange
    client.green1.leds(255, 255, 255)  # blanc
    client.green1.leds(0,   0,   0)    # éteint

    # ── Beep ──────────────────────────────────────────────
    client.green1.beep(440, 200)    # La 440 Hz, 200 ms
    client.green1.beep(523, 300)    # Do, 300 ms
    client.green1.beep(659, 300)    # Mi, 300 ms
    client.green1.beep(784, 500)    # Sol, 500 ms

    # ── Exemple pratique : signaler qu'on a marqué un but ─
    def celebrer_but():
        # LEDs clignotent et bruit de fanfare
        for _ in range(3):
            client.green1.leds(255, 200, 0)   # jaune/or
            client.green1.beep(1047, 150)      # Do aigu
            time.sleep(0.2)
            client.green1.leds(0, 0, 0)
            time.sleep(0.1)

programme_complet_minimal.py
"""
Programme minimal RSK — contrôle les deux robots verts.
Vert 2 défend, Vert 1 attaque.
"""
import rsk
from math import radians

with rsk.Client(host='localhost', key='') as client:
    while True:
        try:
            balle = client.ball
            if balle is None:
                continue        # Balle non visible, rien à faire

            bx, by = balle

            # ── Défenseur (green2) : se placer devant le but ──
            if abs(by) < 0.30:
                client.green2.goto((0.80, by, radians(180)), wait=False)
                # Tirer si la balle est dans la zone de but
                if bx > 0.60:
                    client.green2.kick()
            elif by > 0:
                client.green2.goto((0.80, 0.30, radians(180)), wait=False)
            else:
                client.green2.goto((0.80, -0.30, radians(180)), wait=False)

            # ── Attaquant (green1) : foncer sur la balle ──────
            if client.green1.goto((bx + 0.05, by, radians(180)), wait=False):
                client.green1.kick()

        except Exception:
            pass    # Ignorer les erreurs transitoires

8. Accès dynamique aux robots

Les raccourcis client.green1, client.green2, etc. sont pratiques, mais ils sont figés : la couleur est écrite en dur dans le code. Pour écrire un programme qui s'adapte automatiquement (peu importe si vous jouez en vert ou en bleu), il faut utiliser la syntaxe dynamique :

client.robots dict

Dictionnaire de tous les robots, indexé par couleur puis par numéro.

AccèsÉquivalent raccourci
client.robots['green'][1]client.green1
client.robots['green'][2]client.green2
client.robots['blue'][1]client.blue1
client.robots['blue'][2]client.blue2

Avec cette syntaxe, vous pouvez stocker la couleur dans une variable et ne changer qu'un seul endroit dans votre code pour jouer avec l'autre équipe :

acces_dynamique.py
import rsk

# ← Une seule ligne à changer pour passer en équipe bleue
MA_COULEUR = 'green'

with rsk.Client() as client:
    # Accès par variable : aucun "green" en dur dans le reste du code
    defenseur = client.robots[MA_COULEUR][2]
    attaquant = client.robots[MA_COULEUR][1]

    while True:
        balle = client.ball
        if balle is None:
            continue

        bx, by = balle
        attaquant.goto((bx, by, 0), wait=False)

Vous pouvez aussi parcourir tous vos robots en boucle, utile pour allumer les LEDs ou envoyer une commande à toute l'équipe :

boucle_equipe.py
import rsk

MA_COULEUR = 'green'

with rsk.Client() as client:
    # Allumer tous les robots de l'équipe en rouge
    for numero, robot in client.robots[MA_COULEUR].items():
        robot.leds(255, 0, 0)

    # Ou parcourir TOUTES les couleurs et TOUS les numéros
    for couleur in client.robots:               # 'green', 'blue'
        for numero in client.robots[couleur]:   # 1, 2
            print(couleur, numero, client.robots[couleur][numero].position)
💡 Bonne pratique

Placez toujours MA_COULEUR = 'green' tout en haut du fichier. Ainsi, le reste du code n'a jamais la couleur en dur, et changer d'équipe ne prend qu'une seconde.

9. client.referee — les informations de l'arbitre

Pendant un match, le Game Controller maintient un état de jeu complet. Votre programme peut le lire à tout moment via client.referee, qui est un dictionnaire Python mis à jour en temps réel.

ℹ️ Type : dictionnaire

client.referee peut valoir None si aucun Game Controller n'a encore répondu. Pensez à vérifier avant d'y accéder.

Structure complète du dictionnaire

structure_referee.py (exemple de contenu)
{
    # ── État global du match ─────────────────────────────────────
    "game_is_running":     True,   # Le match a été démarré (bouton "Start")
    "game_paused":         False,  # Jeu en pause (placement de balle, mi-temps…)
    "halftime_is_running": False,  # True pendant la mi-temps
    "timer":               240,    # Temps restant en secondes (arrondi au plafond)
    "game_state_msg":      "Game is running...",  # Message lisible de l'état

    # ── Équipes ──────────────────────────────────────────────────
    "teams": {
        "green": {
            "name":       "Les verts FC",  # Nom de l'équipe (saisi dans le GC)
            "score":      1,               # Score actuel
            "x_positive": True,            # True → vert joue côté X positif (+0.92 m)
            "robots": {
                "1": {
                    "penalized":           False,  # Robot pénalisé ?
                    "penalized_remaining": None,   # Temps de pénalité restant (s) ou None
                    "penalized_reason":    None,   # Raison (ex: "ball_abuse")
                    "preempted":           False,  # Contrôlé par le referee ?
                    "preemption_reasons":  [],     # Liste des raisons de préemption
                },
                "2": { ... }  # Même structure pour le robot 2
            }
        },
        "blue": {
            "name":       "Blue Stars",
            "score":      0,
            "x_positive": False,  # bleu est du côté X négatif (−0.92 m)
            "robots": { "1": { ... }, "2": { ... } }
        }
    },

    # ── Historique des événements arbitraux ──────────────────────
    "referee_history_sliced": [
        "goal",
        "sideline-crossed",
        "penalty-green1",
        ...
    ]
}

Tableau des champs clés

Accès Type Signification
client.referee["game_is_running"] bool Le match a-t-il été lancé (bouton "Start Game") ?
client.referee["game_paused"] bool Jeu momentanément en pause (remise en jeu, mi-temps…)
client.referee["halftime_is_running"] bool True pendant la mi-temps — robots bloqués
client.referee["timer"] int Temps restant en secondes (5 min par défaut = 300 s)
client.referee["game_state_msg"] str Message d'état lisible : "Game is running…", "Place the ball on the dot"…
client.referee["teams"]["green"]["score"] int Score de l'équipe verte
client.referee["teams"]["green"]["name"] str Nom de l'équipe verte (configuré dans le Game Controller)
client.referee["teams"]["green"]["x_positive"] bool Clé cruciale : True → vert joue du côté X + (derrière x ≈ +0.92 m). Voir section suivante.
client.referee["teams"]["green"]["robots"]["1"]["penalized"] bool Le robot vert 1 est-il pénalisé (expulsé temporairement) ?
client.referee["teams"]["green"]["robots"]["1"]["penalized_remaining"] int | None Secondes de pénalité restantes, ou None si pas pénalisé
client.referee["teams"]["green"]["robots"]["1"]["penalized_reason"] str | None Raison : "ball_abuse", "abusive_defense", "abusive_attack"…
client.referee["teams"]["green"]["robots"]["1"]["preempted"] bool Vrai si l'arbitre a pris le contrôle du robot (vous ne pouvez plus le piloter)
client.referee["teams"]["green"]["robots"]["1"]["preemption_reasons"] list[str] Liste des raisons : "game-start", "half-time", "sideline-crossed"…
client.referee["referee_history_sliced"] list[str] Derniers événements arbitraux (buts, sorties, pénalités…)
⚠️ Clés numéros : chaînes de caractères !

Les numéros de robots sont des chaînes dans le JSON : utilisez "1" et "2" (avec guillemets), pas des entiers. 

# ✅ Correct
penalise = client.referee["teams"]["green"]["robots"]["1"]["penalized"]

# ❌ Faux — KeyError !
penalise = client.referee["teams"]["green"]["robots"][1]["penalized"]
lire_arbitre.py
import rsk

with rsk.Client() as client:
    ref = client.referee   # raccourci local
    if ref is None:
        print("Pas encore de données arbitre")
    else:
        print("Match lancé :", ref["game_is_running"])
        print("Temps restant :", ref["timer"], "s")
        print("Score vert :", ref["teams"]["green"]["score"])
        print("Score bleu :", ref["teams"]["blue"]["score"])

        # Vérifier si robot vert 1 est pénalisé
        if ref["teams"]["green"]["robots"]["1"]["penalized"]:
            reste = ref["teams"]["green"]["robots"]["1"]["penalized_remaining"]
            print(f"Robot vert 1 pénalisé — encore {reste}s")

10. Changer de sens automatiquement

Dans le chapitre 5, vous avez deux fichiers séparés (green vers− et green vers+) selon que votre but à défendre est à droite (+X) ou à gauche (−X). Grâce à client.referee et à l'accès dynamique, un seul programme peut détecter automatiquement son côté et adapter tous ses calculs.

Le champ x_positive

L'arbitre maintient, pour chaque équipe, un champ x_positive qui indique de quel côté du terrain elle se trouve en début de match :

x_positive (vert) Position défensive But à défendre Correspond à
True x ≈ +0.80 m (droite) But droit (x ≈ +0.92 m) green vers−
False x ≈ −0.80 m (gauche) But gauche (x ≈ −0.92 m) green vers+
auto_cote.py — programme qui s'adapte tout seul
"""
Programme RSK qui détecte automatiquement son côté grâce à client.referee.
Une seule variable MA_COULEUR à changer pour jouer en bleu.
"""
import rsk
from math import radians, pi

MA_COULEUR = 'green'   # ← seul endroit à modifier

with rsk.Client() as client:

    # ── Attendre les infos de l'arbitre ─────────────────────────
    while client.referee is None:
        pass   # patience le temps de la première mise à jour

    # ── Déterminer le côté une fois pour toutes ──────────────────
    x_positif = client.referee["teams"][MA_COULEUR]["x_positive"]

    if x_positif:
        # Mon équipe est côté X+ → je défends le but de droite
        x_defenseur  =  0.80      # position défensive (en coordonnées terrain)
        angle_face   = radians(180)  # face vers la gauche (vers le terrain)
        signe_attaque = -1           # j'attaque vers X−
    else:
        # Mon équipe est côté X− → je défends le but de gauche
        x_defenseur  = -0.80
        angle_face   = radians(0)    # face vers la droite
        signe_attaque = +1           # j'attaque vers X+

    defenseur = client.robots[MA_COULEUR][2]
    attaquant  = client.robots[MA_COULEUR][1]

    # ── Boucle principale ────────────────────────────────────────
    while True:
        try:
            balle = client.ball
            if balle is None:
                continue

            bx, by = balle

            # Défenseur : se placer devant son but, suivre Y de la balle
            by_clamp = max(-0.30, min(0.30, by))
            defenseur.goto((x_defenseur, by_clamp, angle_face), wait=False)

            # Expulser la balle si elle est trop proche du but
            if abs(bx - x_defenseur) < 0.20:
                defenseur.kick()

            # Attaquant : foncer sur la balle puis tirer
            if attaquant.goto((bx + signe_attaque * 0.05, by, angle_face), wait=False):
                attaquant.kick()

        except Exception:
            pass
💡 Pourquoi signe_attaque * 0.05 ?

On demande au robot de se placer légèrement derrière la balle (côté opposé au but adverse) pour qu'il lui fasse face. Le signe (±1) est déterminé automatiquement selon le côté détecté.

Lire le côté pendant le match (après mi-temps)

Dans certaines compétitions, les équipes changent de côté à la mi-temps. Pour gérer ce cas, il suffit de relire x_positive à chaque tour de boucle (plutôt qu'une seule fois) :

cote_dynamique_boucle.py
import rsk
from math import radians

MA_COULEUR = 'green'

with rsk.Client() as client:
    while True:
        try:
            ref = client.referee
            if ref is None:
                continue

            # Recalculer le côté à chaque itération → mi-temps gérée automatiquement
            x_positif = ref["teams"][MA_COULEUR]["x_positive"]
            x_def   =  0.80 if x_positif else -0.80
            angle   = radians(180) if x_positif else radians(0)
            signe   = -1 if x_positif else +1

            balle = client.ball
            if balle is None:
                continue

            bx, by = balle
            client.robots[MA_COULEUR][2].goto(
                (x_def, max(-0.30, min(0.30, by)), angle), wait=False
            )
            if client.robots[MA_COULEUR][1].goto(
                (bx + signe * 0.05, by, angle), wait=False
            ):
                client.robots[MA_COULEUR][1].kick()

        except Exception:
            pass
