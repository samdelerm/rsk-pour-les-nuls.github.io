RSK : Le terrain

Avant de programmer tes robots, tu dois connaître le terrain comme ta poche. Dimensions, coordonnées, zones, angles — tout est ici.

1. Dimensions du terrain

Le terrain RSK (la zone de jeu) mesure 1,84 m × 1,23 m. Il est posé sur un tapis vert plus grand de 2,45 m × 1,84 m qui laisse une bordure de 30 cm tout autour — nécessaire pour les caméras.

ÉlémentLargeur (X)Hauteur (Y)
Terrain de jeu1,84 m1,23 m
Tapis complet2,45 m1,84 m
Largeur d'un but—0,60 m
Zone de défense0,30 m0,90 m
Robot (diamètre)≈ 7 cm—

2. Système de coordonnées

L'origine (0, 0) est au centre du terrain. L'axe X est horizontal (de gauche à droite), l'axe Y est vertical (de bas en haut). Tout est en mètres.

Le terrain RSK avec le système de coordonnées. L'origine (0,0) est au centre. Tout est en mètres.

Retiens les limites : X ∈ [−0.92, +0.92] et Y ∈ [−0.615, +0.615]. Si tu envoies un robot en dehors de ces valeurs, il sortira du terrain !

constantes_terrain.py
import rsk

# Les constantes sont dans rsk.constants
print(rsk.constants.field_length)    # 1.84 m
print(rsk.constants.field_width)     # 1.23 m
print(rsk.constants.goal_width)      # 0.60 m
print(rsk.constants.defense_area_length)  # 0.30 m
print(rsk.constants.defense_area_width)   # 0.90 m

# Calculer les limites
x_max = rsk.constants.field_length / 2    # 0.92
y_max = rsk.constants.field_width  / 2    # 0.615
print(f"X : [{-x_max:.2f}, {x_max:.2f}]")
print(f"Y : [{-y_max:.3f}, {y_max:.3f}]")

4. Angles & radians

Les angles dans RSK sont exprimés en radians, pas en degrés. Un tour complet = 2π ≈ 6.28 radians.

Le sens positif des angles est le sens anti-horaire (mathématique).

angles.py
import math
from math import radians, pi

# Les angles les plus utilisés dans RSK
print(radians(0))       # 0.0   → regarde vers la droite (+X)
print(radians(180))     # 3.14  → regarde vers la gauche (-X)  ← défenseur vert!
print(radians(90))      # 1.57  → regarde vers le haut (+Y)
print(radians(-90))     # -1.57 → regarde vers le bas (-Y)

# Équivalents avec pi
pi       # ≈ 3.14159 → regarde à gauche

# Dans le code réel :
from math import radians

# Défenseur vert (but à droite → regarde à gauche)
angle_defenseur = radians(180)

# Défenseur vert (but à gauche → regarde à droite)
angle_defenseur_gauche = radians(0)

# Calcul de l'angle vers la balle (math.atan2 → jamais de division par zéro)
xg1, yg1 = 0.46, 0.0     # position du robot
xb, yb   = -0.3, 0.2      # position de la balle

angle_vers_balle = math.atan2(yb - yg1, xb - xg1)
# → L'angle à adopter pour que le robot regarde vers la balle

Les robots RSK utilisent math.atan(dy/dx) dans les codes d'exemple, mais cette forme peut produire une division par zéro si dx == 0. Préfère toujours math.atan2(dy, dx) qui gère tous les cas !

5. La balle

client.ball contient la position de la balle détectée par la caméra. C'est un tuple (x, y) en mètres, ou None si la caméra ne la voit pas.

balle.py
import rsk

with rsk.Client(host='localhost', key='') as client:
    while True:
        # ── Vérification obligatoire ! ─────────────────────
        if client.ball is not None:
            xball = client.ball[0]   # position X de la balle
            yball = client.ball[1]   # position Y de la balle
            print(f"Balle en ({xball:.3f}, {yball:.3f})")

            # Balle dans la moitié gauche du terrain ?
            if xball < 0:
                print("La balle est dans la moitié bleue")
            else:
                print("La balle est dans la moitié verte")

            # Balle en danger pour le but vert (à droite) ?
            if xball > 0.60 and abs(yball) < 0.30:
                print("⚠️ LA BALLE MENACE LE BUT VERT !")

        else:
            print("Balle non visible - attente...")

6. Quelle équipe joue de quel côté ?

RSK peut se jouer dans les deux orientations. Il y a deux configurations correspondant aux deux fichiers green vers-.py et green vers+.py :

green vers− (vert attaque à gauche)

Le but à défendre par le vert est sur la droite (+X) du terrain. Le défenseur se place à x ≈ +0.80, orienté à radians(180) (regarde à gauche). L'attaquant pousse vers x < 0 (vers le but bleu).

green vers+ (vert attaque à droite)

Le but à défendre par le vert est sur la gauche (−X). Le défenseur se place à x ≈ −0.80, orienté à radians(0) (regarde à droite). L'attaquant pousse vers x > 0 (vers le but bleu).

L'arbitre RSK (ou l'organisateur du match) te dit avant la partie si tu joues "vert vers−" ou "vert vers+". Utilise le bon fichier en fonction de cette info !
